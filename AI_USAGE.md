# AI Usage Disclosure

This project was developed with assistance from **Cursor AI** and AI-powered developer tools to improve development speed, UI iteration, debugging, and code quality.

## How AI Was Used

AI assistance was primarily used for:

- UI component generation and layout experimentation
- Debugging and fixing implementation issues
- Improving code structure and developer productivity
- Generating boilerplate and repetitive code patterns
- Refining validation logic and API integration workflows
- Documentation and deployment setup support

## Technologies & Workflows

- React + Vite application setup
- Redux Toolkit state management for multi-step registration flow
- Axios API client configuration with centralized error handling
- Form validation utilities for email, password, OTP, birthday, and profile data
- Protected route handling for sequential onboarding steps
- Vercel deployment configuration and SPA routing support

## Human Design & Engineering Decisions

Key product and implementation decisions were designed and reviewed manually, including:

- Registration flow architecture based on the official **React Coding Challenge** requirements
- User experience decisions for onboarding and navigation flow
- API integration strategy and fallback handling
- Local fallback data for interests and pillars when APIs are unavailable
- Custom styling implementation to align with Woliba branding and provided design assets
- Final review, debugging, testing, and feature integration
- Overall application structure and business logic decisions

AI tools were used as development assistants, while the application architecture, implementation decisions, and final code review remained human-directed.
